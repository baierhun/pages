## Requirements

Your program must:

* Include at least **3 decision branches**
* Use at least **one `and` or `or` statement**
* Clearly explain results to the user
* Have meaningful differences between outcomes
* This must be an **interactive program** — user inputs are required.
* Use an **if/else** or **if/elif/else** structure.
* Include **at least ONE Boolean expression** that uses a **logical operator** (`and`, `or`, `not`).
* Follow good programming style:

  * Program header
  * camelCase variable names
  * Proper spacing
  * Comments
  * Clear input prompts
  * Descriptive output

## ✅ The Process

1. **Prepare**

   * Review the project requirements and coding options.
   * Review the project rubric. Do you understand how projects will be graded?

2. **Plan** your algorithm.
   Use whiteboards, paper, or plan using comments in your code.

3. **Write** the Python code in Lesson 2.6 on CodeHS.

4. **Test** your program.

   * Are you getting the results you expect? Try lots of different inputs.
   * Think about your code from the user's perspective: Are input prompts formatted well?
   * Does your output give clear information?
   * Have a friend try your program and give feedback.

5. **Re-check** the **rubric** to ensure you've completed everything.

6. **Prepare for Code Review Day**

   * Every student will have **2 minutes** to explain their code, followed by **1 minute** of questions from classmates.
   * See the **Code Review** section below for more information on how to prepare.
