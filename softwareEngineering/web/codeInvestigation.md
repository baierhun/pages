# 🕵️ Code Investigation Assignment

## “Figure Out What This Program Does”

### 🔗 Repository

[https://github.com/baierhun/starWars](https://github.com/baierhun/starWars)

---

## ⚠️ Before you begin
* Copy this worksheet into the Google Doc connected to the assignment on Canvas

## 🎯 Objective

You are given a codebase with **little explanation**.

Your job is to:

* Determine what the program does
* Understand how it works
* Identify its structure and components
* Support every claim with **evidence from the code**

You are not expected to understand everything. You are expected to **investigate thoroughly**.

---

## 🧠 Rules of the Investigation

* Do NOT guess without evidence
* Every claim should reference:

  * a file
  * a line of code
  * or a specific behavior
* It’s okay to be wrong — but explain your reasoning
* If you’re unsure, say what you *think* and *why*

---

# 📝 Investigation Worksheet

## 🧩 Part 1: First Look

1. What is your first guess about what this program does?

2. What specific clues led you to that guess?

3. What is still unclear after your first look?

---

## 📁 Part 2: File & Folder Structure

4. List all top-level files and folders in the repository.

5. Which files seem most important? Why?

6. Are there any files that look like:

   * an entry point (where the program starts)?
   * configuration files?
   * data files?

Explain your reasoning.

---

## ⚙️ Part 3: Language & Tools

7. What programming language(s) are used?

8. How do you know? (file extensions, syntax, etc.)

9. Are there any external libraries or dependencies?

   * Where are they defined?

10. Is there anything that looks like a build or run system?

---

## ▶️ Part 4: Running the Program

11. Are there instructions for running the program?

12. If not, how would you try to run it?

13. What command(s) might work?

14. What do you expect to happen when it runs?

---

## 🔍 Part 5: Code Exploration

15. Open what you think is the **main file**:

* What does it do?

16. Find a function or method:

* Name:
* What it appears to do:
* What inputs it takes:
* What outputs it produces:

17. Are there any variables that seem important?

* What are they used for?

---

## 🔄 Part 6: Program Behavior

18. Step through the program (mentally or by running it):

* What happens first?
* What happens next?

19. Does the program:

* take input?
* produce output?
* interact with a user?

Explain how.

---

## 🌐 Part 7: Data & External Connections

20. Does the program use any data?

* Where is the data stored?

21. Does it connect to anything outside the program?

* APIs?
* files?
* databases?

Provide evidence.

---

## 🧠 Part 8: Putting It Together

22. What is the purpose of this program?

23. Who would use it?

24. What problem does it solve?

---

## ⚠️ Part 9: Confusion & Questions

25. What parts of the code are confusing?

26. What questions would you ask the original developer?

---

## 🔧 Part 10: Improvements

27. What would you improve about this program?

28. What feature would you add?

---

## 🧾 Final Summary

29. In 3–5 sentences, explain what this program does.

30. What evidence supports your explanation?

---

* You may work with a partner, but answers must be your own
